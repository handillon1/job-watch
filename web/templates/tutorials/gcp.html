Create a GCP project + service account
  1. Go to Google Cloud Console and sign in with the Google account that owns your sheet.
  2. Top bar: project selector → New Project. Pick any name (e.g. job-watch). Click Create.
  3. Once the project is created, switch to it via the project selector.
  4. In the top search bar, search "Sheets API" → click the result → Enable. This is required or sheet writes fail with HTTP 403.
  5. Left nav: IAM & Admin → Service Accounts → + Create service account.
  6. Service account name: anything (e.g. job-watch-sheet-writer). Skip the "grant access" step. Click Done.
  7. Click into the service account → Keys tab → Add Key → Create new key → JSON → Create. A JSON file downloads automatically.
  8. Open the JSON file in a text editor and copy the entire contents. Add it as the GOOGLE_SERVICE_ACCOUNT_JSON GitHub secret (paste the whole JSON as the value).
  9. The client_email field in the JSON is the address you'll share your sheet with (see the Google Sheets card → "Share with service account" tutorial).
Verify the Sheets API is enabled
  1. Open the direct link to the Sheets API page for your project: https://console.cloud.google.com/apis/library/sheets.googleapis.com?project=<your-project-id> (replace <your-project-id> with the value shown in the GCP service account card details above).
  2. If the button says Enable, click it.
  3. If it says Manage or shows "API enabled", you're set.
  4. Refresh this dashboard — the Sheets card should be green.
Rotate the service account key (after a leak)

If your GOOGLE_SERVICE_ACCOUNT_JSON leaks (e.g. accidentally committed, shared in chat), do this immediately:

  1. In Google Cloud Console: IAM & Admin → Service Accounts → click your SA → Keys tab.
  2. Find the leaked key (match by private_key_id in the JSON). Click the trash icon → Delete.
  3. Create a new key: Add Key → Create new key → JSON → Create. A new JSON downloads.
  4. Update the GOOGLE_SERVICE_ACCOUNT_JSON GitHub secret with the new JSON content.
  5. Update your local web/.env or web/.sa.json file too.
  6. The next workflow run will use the new key automatically. The old one is dead immediately.
Switch to a different sheet / service account
  1. If switching sheets: get the new sheet's ID from its URL, update APPLICATIONS_SHEET_ID GitHub secret.
  2. If switching service accounts: create the new SA (see "Create a GCP project + service account" above), download its JSON, update GOOGLE_SERVICE_ACCOUNT_JSON.
  3. Either way: Share the new sheet with the SA email (Editor permission). The dashboard's GCP card always shows the current SA email.
  4. If the new sheet doesn't have an Applications tab, the bot writes to the first tab. Make sure the schema matches (Company, Role, Date Applied, Email Applied, Resource, Status).
