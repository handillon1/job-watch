Create a new Telegram bot
  1. Install Telegram on your phone or use Telegram Web.
  2. In the search bar, find @BotFather (verified blue checkmark).
  3. Send /newbot and follow the prompts: pick a display name, then a unique username ending in bot.
  4. BotFather replies with your token — looks like 1234567890:ABC.... Save it; you'll add it as the TELEGRAM_BOT_TOKEN secret next.
  5. In Telegram, search for your new bot by username and send it any message (e.g. "hi"). This is required so the bot can DM you back.
Find your Telegram chat ID
  1. Make sure you've sent your bot at least one message first (see tutorial above).
  2. Open a terminal and run, replacing <YOUR_TOKEN>: 
    curl "https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates"
  3. Look in the JSON response for "chat":{"id":<number>. That number is your TELEGRAM_CHAT_ID — usually 9-10 digits, positive for a personal DM.
  4. If the response is {"ok":true,"result":[]}, you didn't message your bot first. Send it any message and re-run.
Update the bot token or chat ID
  1. Generate the new value (re-create the bot, or look up your chat ID again).
  2. Go to GitHub → Settings → Secrets and variables → Actions.
  3. Click the secret name (TELEGRAM_BOT_TOKEN or TELEGRAM_CHAT_ID) → Update → paste the new value → Update secret.
  4. Also update your local web/.env if you want the dashboard to use the new value.
  5. The next workflow run (within ~1 min via cron-job.org) will pick up the new value automatically — no code changes needed.
Disable a stuck webhook

If the dashboard shows a webhook URL set, getUpdates will return HTTP 409 and the apply-button flow breaks. The workflow auto-clears the webhook on every run as a safety net, but you can also clear it manually:

curl "https://api.telegram.org/bot<YOUR_TOKEN>/deleteWebhook"

Then refresh the dashboard — the Telegram card should clear back to green.
